# 🎨 UI/UX 디자인 시스템 — 영농형 태양광 관리

> 웹 버전(`web_app.py` + `templates/` + `static/style.css`)의 디자인 규칙 문서입니다.
> 콘셉트는 **“밝은 미니멀 (iOS풍)”** — 흰 배경, 넓은 여백, 헤어라인 구분선, 정보 중심.
> 화면을 새로 만들거나 고칠 때 이 문서의 토큰·컴포넌트를 그대로 재사용하세요.

---

## 1. 디자인 원칙

1. **정보가 주인공** — 장식 최소화. 큰 숫자와 또렷한 위계로 한눈에 읽히게.
2. **iOS 그룹 리스트 문법** — 시스템 회색 배경 위에 흰 둥근 카드, 카드 안은 인셋 헤어라인으로 행을 구분.
3. **색은 의미일 때만** — 그린은 강조(틴트) 1색. 금액은 관습대로 **수입 파랑 / 지출 빨강**.
4. **모바일 우선** — 농가에서 휴대폰으로 쓰는 상황 기준. 탭 영역 ≥ 50px, 가로 스크롤 표 대신 세로 리스트.
5. **본문 16px 고정** — iOS에서 입력 포커스 시 화면 확대(zoom)되지 않는 최소 크기.

## 3. 형태 · 타이포 · 레이아웃

### 모서리 / 반경
- `--radius: 16px` — 카드·폼·요약·연도선택
- `--radius-sm: 11px` — 작은 요소
- `--pill: 999px` — 칩·뱃지·메뉴 버튼

### 타이포그래피
- 글꼴: `-apple-system, BlinkMacSystemFont, Pretendard, Apple SD Gothic Neo, Noto Sans KR …`
- 본문 16px / `line-height 1.5` / `letter-spacing -.014em`
- **큰 타이틀** `.lt` — 1.95rem / 800 / `-.035em` (각 페이지 상단 1개)
- **섹션 헤더** `.sec` — 0.82rem / 600 / `--label-3` (카드 위 회색 라벨)
- **요약 숫자** `.summary .big` — 2.3rem / 800
- 숫자는 `.num` 클래스로 **tabular-nums** 고정폭 정렬

### 레이아웃 그리드
- 콘텐츠 최대폭 `--maxw`: 모바일 **580px**, `≥641px`에서 **700px**, 가운데 정렬
- 좌우 패딩 16px
- 상단 `.nav`(48px) 고정 + 하단 `.tabbar` 고정. 본문은 `padding-bottom`으로 탭바 높이만큼 확보.
- `env(safe-area-inset-*)`로 노치/홈바 대응.

## 4. 컴포넌트

### 4.1 상단 내비바 `.nav`
- 반투명(`rgba(248,248,250,.82)`) + `backdrop-filter blur(18px)`, 하단 0.5px 헤어라인.
- 좌: `.nav-title`(농장 이름 또는 기본 타이틀), 우: `.more`(⋯ 메뉴, `<details>` 팝오버).
- 메뉴 항목: 데이터 내보내기 / 예시 데이터 / 전체 초기화(위험=빨강) / 로그아웃(인증 시).

### 4.2 하단 탭바 `.tabbar`
- 반투명 + 블러, 상단 헤어라인. 6개 탭(홈·농축산·태양광·재무·통계·서류).
- 라인 아이콘(26px) + 0.66rem 라벨. 활성 탭은 `.active`로 `--tint` 색.
- 아이콘은 `base.html`의 `icon()` 매크로(SF 심볼풍 stroke SVG).

### 4.3 그룹 카드 `.card` + 행 `.row`
iOS 인셋 그룹 리스트의 핵심 단위.
```html
<div class="sec">섹션 제목</div>
<div class="card">
  <div class="row">
    <div class="ic">🌾</div>            <!-- 선택: 리딩 아이콘(.solar/.common 변형) -->
    <div class="main">
      <div class="title">주제목 <span class="badge on">진행중</span></div>
      <div class="sub">보조 설명 · 메타</div>
    </div>
    <div class="val num">123<span class="u">kWh</span></div>  <!-- 우측 값 -->
    <div class="acts">…수정/삭제 버튼…</div>
  </div>
</div>
```
- 행 사이 구분선은 `.row::after`(왼쪽 16px 인셋). 아이콘 정렬용 `.row.inset`은 56px 인셋.
- 빈 상태는 `<div class="empty">…</div>`.

### 4.4 요약 카드 `.summary`
대시보드·재무 상단의 핵심 지표.
- `.cap`(라벨) → `.big.plus|.minus`(큰 숫자, +녹/−적) → `.duo`(수입/지출 2분할).
- `.duo .col.in / .col.out` 각 컬럼은 점(•) 색으로 파랑/빨강 구분.

### 4.5 폼 `.form`
- 카드형 인셋 그룹. 각 `<label>` = 한 행.
  - **인라인**(기본): 라벨 좌 · 입력 우측 정렬 — 선택/날짜/숫자 등 짧은 값.
  - **스택**(`.stack`): 작은 회색 라벨 위 + 전체폭 입력 아래 — 품목명·항목·메모 등 텍스트.
- 입력은 **테두리 없음**(borderless), 행 헤어라인으로만 구분. `<select>`는 우측 chevron.
- 보조 설명은 `.form-note`(카드 하단, 옅은 면).
- 제출 버튼은 카드 **밖** `.actions`에 전체폭으로.

### 4.6 버튼
| 클래스 | 모양 | 용도 |
|--------|------|------|
| `.btn.btn-primary` | 그린 채움 | 주요 동작(저장/추가) |
| `.btn.btn-accent` | 골드 채움 | 태양광 ‘발전수익 재무 반영’ |
| `.btn.btn-ghost` | 연한 채움+틴트 글자 | 취소 등 보조 |
| `.btn.btn-danger` | 연한 빨강 | 위험 동작 |
| `.btn-mini` / `.btn-mini.del` | 작은 알약 | 행 안의 수정/삭제 |
| `.icon-btn` / `.icon-btn.del` | 34px 정사각 아이콘 | 재무 행의 수정/삭제 아이콘 |
- `.block`을 붙이면 전체폭. 최소 높이 50px(미니 34px).

### 4.7 뱃지 · 칩 · 자동표시
- `.badge.on`(진행중=그린) / `.badge.off`(종료=회색)
- `.chip` — 진행중 품목 요약(알약, 틴트 소프트)
- `.auto` — 자동 생성 항목(발전 정산금) 표시(골드 소형 태그)

### 4.8 금액 표기
- `.amt-in`(파랑, `+`) / `.amt-out`(빨강, `−`)
- 요약 큰 숫자: `.big.plus`(녹) / `.big.minus`(적), 누계 순이익은 `.val.plus/.minus`.

### 4.9 차트 (`web_charts.py`)
- 외부 라이브러리 없이 **서버에서 SVG 문자열** 생성 → `{{ chart|safe }}`로 삽입.
- `.chart-wrap` 안에서 `width:100%` 반응형. 막대(수입/지출)·꺾은선(발전량 추이).
- 데이터 없으면 “기록이 없습니다” 플레이스홀더.

### 4.10 날짜 그룹 헤더 `.day` (재무)
- `.d`(6월 15일) · `.w`(요일 칩, 토=파랑/일=빨강) · `.net`(그날 합계, 우측).
- 필터: `|md`(6월 15일), `|wday`(요일) — `web_app.py`에 정의.

### 4.11 로그인 `.login`
- 가운데 단일 카드 `.login-card` + 그라데이션 로고 `.login-logo` + 비밀번호 입력 + 그린 버튼.
- `APP_PASSWORD` 환경변수가 있을 때만 노출(로컬 모드는 비표시).

## 6. 접근성 · 인터랙션

- **탭 영역** 최소 ~50px, 아이콘 버튼 34px. 손가락 입력 여유.
- **대비**: 본문/배경, 금액 색은 흰 배경에서 WCAG AA 수준 대비 유지.
- **포커스**: 입력 포커스 시 캐럿 표시(테두리리스), 로그인 입력은 틴트 글로우.
- **모션**: 버튼 `:active` 시 0.5px 눌림 + 살짝 밝기 변화만(과한 애니메이션 없음).
- **입력 키패드**: 숫자 필드 `inputmode="numeric|decimal"`, 날짜 `type="date"`.
- **JS 의존 최소**: 메뉴 닫기·서비스워커 등록만(`app.js`). 폼은 순수 POST로 동작(JS 꺼져도 사용 가능).

## 7. 반응형 · PWA

- 한 컬럼 모바일 레이아웃을 기본으로, `≥641px`에서 콘텐츠 폭만 700px로 확장(데이터 밀도 유지).
- **PWA**: `manifest.webmanifest`(테마 `#f8f8fa`, 배경 `#f2f2f7`), `sw.js`로 홈 화면 설치·오프라인 캐시.
- 상태바 색(`<meta name="theme-color">`)은 상단바와 동일한 `#f8f8fa`로 통일.

## 8. 수정 가이드 (자주 하는 변경)

- **강조색 변경**: `--tint` 계열 3개 + `web_charts.py`의 `SOLAR`를 함께 수정.
- **새 카드 섹션 추가**: `<div class="sec">제목</div>` + `<div class="card"> … .row … </div>` 패턴 복사.
- **새 입력 폼**: `.form` 카드 안에 `<label>`(짧은 값) 또는 `<label class="stack">`(텍스트) 추가 → 버튼은 `.actions`.
- **서류 링크 추가**: `web_app.py`의 `LINK_GROUPS`만 수정(템플릿/CSS 불필요).
- **금액/숫자 표시**: 통화는 `won()`, 축약은 `won_short()`(context_processor로 전 템플릿 사용 가능).

## 9. 파일 맵

```
static/style.css   ← 디자인 토큰(:root) + 모든 컴포넌트 스타일(이 문서의 단일 원천)
static/app.js      ← 메뉴 닫기 + 서비스워커 등록
static/manifest.webmanifest, sw.js, icon-*.png  ← PWA
templates/base.html        ← 상단바·탭바·아이콘 매크로·플래시(공통 골격)
templates/<화면>.html      ← 각 탭 화면 (위 5장 표 참고)
web_charts.py      ← SVG 차트 + 통화 포맷(색상은 style.css와 동기화)
```

> 디자인을 바꿀 때는 **개별 템플릿이 아니라 `style.css`의 토큰·컴포넌트**를 먼저 고치세요.
> 클래스 이름은 템플릿과 CSS가 1:1로 묶여 있으므로, 새 클래스를 만들기보다 기존 컴포넌트를 재사용하는 것을 권장합니다.
